# 기여로

기여로는 오픈소스에 처음 참여하는 사람이 자신에게 맞는 프로젝트와 이슈를 찾고, 실제 기여를 시작할 수 있도록 돕는 서비스입니다.

GitHub에는 수많은 프로젝트와 이슈가 있지만 처음 기여하는 사람은 어떤 저장소가 계속 관리되고 있는지, 어떤 작업부터 시작해야 하는지 판단하기 어렵습니다. 기여로는 사용자의 경험과 저장소의 최신 상태를 함께 분석해 선택지를 줄이고, 기여 과정을 단계적으로 안내합니다.

## 기여로가 해결하려는 문제

- 어떤 오픈소스 프로젝트를 선택해야 할지 판단하기 어렵습니다.
- 이슈가 많아 자신의 경험과 작업 범위에 맞는 일을 찾기 어렵습니다.
- 저장소가 현재도 관리되는지, 외부 기여를 잘 받아주는지 직접 확인해야 합니다.
- 이슈를 찾은 뒤에도 기여 절차와 코드베이스를 이해하는 데 시간이 필요합니다.

## 주요 기능

- **맞춤 프로젝트 추천**: GitHub에서 자주 사용한 언어, 최근 공개 활동, 참여 프로젝트의 공개 의존성을 분석해 새로운 오픈소스 프로젝트를 추천합니다.
- **단계별 첫 기여 탐색**: 문서 번역에서 시작해 테스트 코드·타입 개선, 버그 수정·기능 추가로 이어지는 성장 경로를 제공합니다.
- **기여하기 좋은 이슈 선별**: 담당자와 연관 Pull Request가 없는 이슈를 우선 확인하고, 난이도와 작업 유형을 함께 보여줍니다.
- **저장소 상태 확인**: 최근 개발 활동, 기여 가이드, 외부 Pull Request에 대한 관리자 응답을 바탕으로 저장소의 기여 친화도를 보여줍니다.
- **이슈 이해 지원**: GitHub 이슈의 핵심 내용, 예상 작업 범위와 주의점을 한국어로 정리합니다.
- **기여 준비와 진행 관리**: 저장소별 기여 가이드를 확인하고 관심·진행 중·완료 작업을 마이페이지에서 관리합니다.

## 추천 방식

기여로는 단순한 인기나 트렌드보다 사용자가 실제로 시작할 수 있는지를 우선합니다. 사용 경험과 기술 스택의 유사도뿐 아니라 저장소가 최근에도 관리되고 있는지, 기여 절차가 준비되어 있는지, 입문자가 맡을 만한 열린 이슈가 있는지를 함께 확인합니다.

추천 결과에는 판단 근거를 함께 표시해 사용자가 프로젝트를 직접 비교하고 선택할 수 있도록 합니다.

## 데이터와 권한

맞춤 추천은 공개 GitHub 저장소와 공개 활동만 사용합니다. 비공개 저장소 권한은 요청하지 않으며, 마이페이지의 작업 목록은 현재 사용 중인 브라우저에 저장됩니다.
